My Game

This is the game I have been slowly working on for a few months now, I have created everything from code to art. Before I started on this game I only had about a month of game dev experience in Unity, I have picked up C# quite quickly due to knowing a different language, Visual Basic, previously.

To connect rooms in my game I opted for a very simple level manager, it uses scriptable objects to connect scenes. This system is a reliable and easily expandable way to create scene transitions and I continue to create rooms.

The player is not able to interact with anything when enemies are nearby, this is done with a simple trigger collider. You may also realise the character says different things in both these clips, when creating the dialogue system I didn't want dialogue options but I also didn't want dialogue to be boring. This is a small example of the NPC reacting to the players health, in the future I can expand on this easily with the system I created.

Keeping the scope of the game small when creating a metroidvania is definitely tricky, I plan on creating one area at a time to easily limit scope, this and, for me i'm creating this game more for fun and a learning experience rather than creating a final product.

The player and all the enemies use finite state machines, with a state machine behaviour is split up into distinct stats, for example running, idling or attacking. States are connected by transitions, which allow the enemy or player to move between states based on certain conditions like a button press.They are also really good for the efficiency of code as everything is organised into distinct states meaning it's easier to manage and debug. It also means I can create new enemies quicker as I don't have to rewrite any code for the states I've already created.

Map / Level Design

As the game is a metroidvania it will have a large and interconnected map, rather than loading a single large scene I decided to split the map into different rooms. This allows for fast loading times and a high framerate at all times. To keep track of all the levels I'm using a tool called Level Designer Toolkit, an open source map making level editor from the creator of Dead Cells. In LDtk I block each level so I can visualise it and know what levels it's going to be connected to, I then import them as PNG's into Unity and go over them with the tilesets I have created. I use Unitys auto tilemap feature which allows you to create rules for tiles in your scene and how they should connect, I can then quickly populate the level with decorations and enemies. Once I have created the art for a level, building it is relatively quick with this method.
